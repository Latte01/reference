## 基本信息

`基本信息` 部分如上图所示，用户根据实际情况进行填写，其中“可见性”仅管理员组的成员可使用。

可见性说明：

| 可见性          | 说明                                                      |
| -------------- | -------------------------------------------------------- |
| 私有            | 知识库仅管理员组内可见，其他组不可见                           |
| 完全公开        | 公开知识库，平台内均可见                                      |
| 仅详情页可见     | 对于管理员组以外的其他组，仅可见知识库的详情页，知识库内的文件不可见 |

## 能力配置

字段说明：
| 字段         | 说明                          |
| ----------  | ----------------------------- |
| 类型        | 必填，指的是技能提供类型，当前只提供 `接口声明` 类型 |
| 认证方式     | 必填，即使用技能的认证方式，需要根据技能的实际使用方式进行设置，以便可以成功使用该技能 |
| OpenAPI Schema | 必填，`接口声明` 类型技能的配置方式，需要导入一个 `OpenAPI Schema` 来添加技能|

## OpenAPI Schema

为了让大模型能更好地识别和使用您的技能，您需要提供技能的OpenAPI描述文件，示例如下所示。你可以前往 OpenAPI 官网了解更多 OpenAPI 相关信息。

    ```yaml Example.yaml
    openapi: 3.0.1
    info:
        title: 天气预报
        description: 个性化查询不同地区的天气。
        version: "v1"
    servers:
        - url: http://127.0.0.1:8081
    paths:
        /get_weather:
            post:
            operationId: getWeather
            summary: 展示天气
            requestBody:
                required: true
                content:
                application/json:
                    schema:
                    $ref: "#/components/schemas/getWeather"
            responses:
                "200":
                description: 天气展示完成
                content:
                    application/json:
                    schema:
                        $ref: "#/components/schemas/weatherResponse"
    components:
    schemas:
        getWeather:
        type: object
        required: [city]
        properties:
            city:
            type: string
            description: 城市
        weatherResponse:
        type: object
        required: [weather]
        properties:
            weather:
            type: object
            description: 天气内容
    ```
